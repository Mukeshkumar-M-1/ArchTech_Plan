Phase 02 · stages 2.1 – 2.6

Memory Creation Pipeline

Turn the flat requirements catalog into a structured knowledge base — the Memory Store.

How it works

The AI takes each raw rule and expands it into a detailed knowledge sheet. It then groups them by category, builds summaries, and maps the connections between different systems, creating a complete "digital brain" for the project.

2.1

Requirements Enrichment

Expands short, raw rules into detailed, easy-to-read knowledge sheets.

What this stage is doing

  • Takes each basic rule from Phase 1 and uses the AI to write a comprehensive markdown page about it.
  • It keeps the original rule, but adds clear explanations, priority levels, and related keywords.

Why it is useful

Turns boring technical data into rich, readable context that other AI agents can easily understand when writing the final document.

Problem Solved

The Lack of Context Problem: Basic rules are hard to understand on their own. This stage adds all the missing background information.

Data State ENRICHED 
# REQ-HW-INT-042
**Rule:** The system shall support SPI, 10MHz.
**Priority:** High
**Context:** This is required for high-speed synchronous communication with the external flash memory.
2.2

Category Summaries

Groups rules by department (like Power or Clocks) and writes a high-level overview.

What this stage is doing

  • Bundles all the rules that belong to a major engineering category together.
  • The AI then writes a professional summary explaining what that entire category is about.

Why it is useful

Creates perfect introduction paragraphs for the major chapters of your final report.

Problem Solved

The Information Fragmentation Problem: You no longer have to read 50 individual rules to understand how the Power system works; you just read the summary.

Data State SUMMARIZED 
# Category: Hardware Interfaces
This section defines the communication protocols (SPI, I2C, UART) required for the system. 
Includes 14 individual requirements...
2.3

Sub-category Summaries

Drills down deeper, grouping rules into highly specific subgroups.

What this stage is doing

  • Takes a major category (like "Interfaces") and groups the rules into specific sub-topics (like "UART", "SPI", or "I2C").
  • Generates a targeted, specific summary for each sub-topic.

Why it is useful

Helps organize the final document into logical, easy-to-read subsections without mixing different topics together.

Problem Solved

The Mixed-Topic Problem: Automatically keeps different protocols and modules strictly separated so the final report isn't a confusing mess.

Data State GROUPED 
## Sub-Category: SPI Protocol
The SPI bus operates at 10MHz and acts as the primary link to external flash storage.
Requirements: REQ-042, REQ-043
2.4

Relationship Mapping

Connects the dots by finding rules that affect each other.

What this stage is doing

  • The AI scans every rule and maps out connections (e.g., if a processor speed rule affects a memory timing rule).
  • Creates a master map of all engineering dependencies.

Why it is useful

Shows engineers exactly what will break if they change a specific rule.

Problem Solved

The Isolated Rule Problem: In complex projects, rules are deeply connected but scattered across pages. This stage links them together like a web.

Data State MAPPED 
{
  "source": "REQ-HW-INT-042 (SPI 10MHz)",
  "impacts": ["REQ-PWR-012 (3.3V Logic Level)"],
  "reason": "SPI bus speed dictates the required IO voltage switching limits."
}
2.5

Project Overview

Looks at the entire project and writes a high-level executive summary.

What this stage is doing

Analyzes everything created so far and drafts a global summary of the project's goals, scale, and technical constraints.

Why it is useful

Creates the perfect "Executive Summary" or "Introduction" for the final document instantly.

Problem Solved

The Blank Page Problem: Writers no longer have to spend hours trying to summarize a 500-page document; the AI writes the perfect summary automatically.

Data State SYNTHESIZED 
# Project Overview
This project defines a high-speed embedded controller featuring 14 interface protocols, strict power management constraints, and 124 total hardware requirements.
2.6

Update Index

Builds a master "Table of Contents" so the AI can find anything instantly.

What this stage is doing

Compiles a massive directory list of every single rule, summary, and map created in the previous steps.

Why it is useful

Acts as the central map for the entire knowledge base. When the AI needs to write a specific chapter later, it uses this index to find the exact files it needs.

Problem Solved

The Lost File Problem: With hundreds of generated files, things get messy. This index guarantees the AI never loses track of a single detail.
