Phase 07 · stages 7.1 – 7.3

Template User Clarification

Interactively ask the user for whatever specific details are still missing.

678 part of the verification loop · repeats until no gaps remain
How it works

When Phase 6 detects that the system cannot autonomously complete the document due to missing data, this phase pauses the automation and engages the human expert to provide the missing pieces.

7.1

Generate Missing Questions

Translates technical gaps into clear, human-readable questions.

What this stage is doing

Takes the raw gap logs (like MISSING: clock_speed_hz) and uses an LLM to rewrite them into friendly, understandable questions like: "What is the primary system clock speed in Hz?"

Why it is useful

Prevents the user from having to interpret raw JSON logs or system variable names.

Problem Solved

The Technical Translation Problem: Makes the system accessible to project managers or non-developers who are filling out the forms.

Data State GENERATED 
{
  "gap_id": "ERR_042",
  "raw_log": "MISSING: memory/system/clock_speed_hz",
  "human_question": "What is the primary system clock speed in Hz?"
}
7.2

Ask User

Presents the questions via a chat interface or form.

What this stage is doing

Halts the automated pipeline and presents the questionnaire to the user. It uses Progressive Disclosure to ask questions in manageable batches.

Why it is useful

Establishes the interactive, collaborative nature of the AI agent.

Problem Solved

The Guessing Game Problem: Prevents the AI from hallucinating or guessing critical hardware specifications when data is missing.

Data State WAITING 
// UI Payload
{
  "prompt": "User Input Required",
  "batch_size": 5,
  "remaining_questions": 12,
  "strategy": "PROGRESSIVE_DISCLOSURE",
  "status": "PAUSED_FOR_HUMAN"
}
7.3

Collect Answers

Captures the user's input and readies it for the memory store.

What this stage is doing

Captures the free-text responses from the user, performs basic sanitization, and structures them into a payload.

Why it is useful

Prepares the messy human input into a format that Phase 8 can inject directly into the markdown memory files.

Problem Solved

The Input Formatting Problem: Cleans up typos or conversational text before saving it.

Data State COLLECTED 
{
  "user_input": "It should be around 100MHz I think.",
  "sanitized_value": "100MHz",
  "target_memory_key": "clock_speed_hz",
  "status": "READY_FOR_MEMORY"
}
